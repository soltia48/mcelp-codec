# 三菱CELP — 技術ドキュメント

> English: [../en/README.md](../en/README.md)

このディレクトリには、本クレートで実装されている **三菱CELP方式音声コーデック**
(Mitsubishi CELP speech codec) の技術的な説明をまとめています。
本実装は既存のデコーダをリバースエンジニアリングして得られたものであり、
このドキュメントもそのリバース挙動から推測されたフォーマット・アルゴリズムの
記述であって、公式仕様の公開ではありません。

## 概要

| 項目                  | 値                                                                |
| --------------------- | ----------------------------------------------------------------- |
| サンプリングレート    | 8 kHz, モノラル                                                   |
| 出力コンパンディング  | μ-law (G.711)                                                     |
| フレーム長            | 40 ms = 320 サンプル                                              |
| 圧縮フレームサイズ    | 18 バイト (144 ビット, 138 ビットのペイロード + 1 制御フラグ)     |
| ビットレート          | 18 B / 40 ms ≈ **3.6 kbit/s** (ペイロードのみで 3.45 kbit/s)      |
| LPC 次数              | 10                                                                |
| サブフレーム          | 4 × 80 サンプル (= 各 10 ms), 2 ブロックにグループ化              |
| LSF 量子化            | 2 段階 split-VQ + フレーム間予測 (3 フレーム)                     |
| 適応コードブック      | 分数ピッチ (1/3 サンプル分解能), 10-tap 補間                      |
| 固定コードブック      | 代数的, 6 lag クラス — メイン 5 経路 + ショート 1 経路            |
| ゲイン量子化          | 7 bit 位相インデックス + 4-tap 予測履歴                           |
| ポストフィルタ        | ハーフフレーム単位の前向き + 後ろ向き formant/tilt IIR            |

## 表記規約

各ドキュメントを通じて、以下の規約を用います。

- **Q フォーマット.** `Qn` は小数部 `n` ビットの固定小数点フォーマットを表します。
  スペクトル状態の大半は **Q15** (符号付き、範囲 ≈ [-1, +1)) で保持され、
  LPC 係数は **Q12** で出力されます。内部のアキュムレータ経路では
  **Q31 / Q40** (32-bit / 40-bit 符号付きアキュムレータを `i64` 上で実装) を
  使用します。固定小数点演算の基本要素は [src/arith.rs](../../src/arith.rs)
  にあります。
- **40 ビットアキュムレータ (i40).** 元の DSP は 40-bit MAC アーキテクチャです。
  いくつかの算術ヘルパ (`shift_acc40`, `to_i40`, `exp_acc`,
  `norm_acc_with_t`, `acc32_16`) が i40 のラップ / 符号拡張セマンティクスを
  忠実に再現しています。これらがあるおかげで、本コーデックは
  リバース対象のリファレンスとビット単位で一致します。
- **エンディアン.** 144 ビットフレームは MSB ファーストで読み出されます
  ([bitstream](architecture.md#3-ビットストリーム) 参照)。
- **サブフレーム / ブロック.** *サブフレーム* は 80 サンプルです。
  連続する 2 つのサブフレームが *ブロック* (= 160 サンプルのハーフフレーム)
  を形成します。1 フレームには 2 ブロックが含まれます。
